# Codex / Remote Code UI Gap Analysis

Дата ревизии: 2026-05-12.

## Главная продуктовая цель

Remote Code должен ощущаться как мобильное и VS Code-зеркало Codex Desktop, а не как отдельный чат. Пользователь должен видеть текущий проект, список чатов именно этого проекта, ход работы модели, действия, файлы, проверки и обновление приложения без ручной настройки портов.

## Что уже совпадает с Codex Desktop

| Зона | Codex Desktop | VS Code extension | Android app |
| --- | --- | --- | --- |
| Чат | Основной рабочий экран | Есть Remote Code чат в webview | Есть Codex чат |
| История чатов | Есть список сессий | Есть список чатов в sidebar | Есть диалог истории чатов |
| Проектный контекст | Сессия имеет `cwd` | Workspace доступен из VS Code | Папки доступны через вкладку проектов |
| Модель/effort | Выбор модели и усилия | Есть модель, effort, profile, context toggle | Есть модель, effort, profile, context toggle |
| Видимый прогресс | Есть поток статусов и действий | Добавлены `model_progress` события | Показываются `model_progress` события |
| Подтверждения | Команды/правки требуют action flow | Есть approve/deny | Есть approve/deny |
| Вложения | Поддерживаются файлы/изображения | Есть attachment flow | Есть file picker и voice input |
| Файлы | Можно работать с проектом | Есть workspace tree/read-file | Есть вкладка проектов/файлов/просмотра |

## Ключевые расхождения

### 1. Проекты и чаты

Codex Desktop привязывает сессию к `cwd`; пользователь естественно мыслит "чат в проекте". Расширение до этой ревизии показывало проекты отдельно и чаты отдельно. Android показывал проекты глубже, внутри вкладки файлов, поэтому главный экран выглядел как "только чаты".

Что исправлено сейчас:
- thread summaries расширения получают `workspaceName` и `workspacePath`;
- Codex session `.jsonl` читается до `session_meta.cwd`, чтобы чаты Codex Desktop тоже имели проект;
- sidebar расширения показывает проект под названием чата;
- Android `CodexThread` принимает `workspaceName/workspacePath`;
- Android история чатов показывает проект в subtitle;
- верхняя вкладка Android переименована с "ФАЙЛЫ" на "ПРОЕКТЫ".
- Android верхняя панель чата получила постоянный селектор проекта;
- выбор проекта на Android переключает последний чат проекта или оставляет пустую рабочую область для нового чата в выбранном проекте;
- Android сохраняет выбранный проект в `SharedPreferences` и восстанавливает его после перезапуска;
- Android drawer получил поиск по проектам и чатам;
- новый чат в расширении наследует проект текущего выбранного чата, если проект не передан явно через API.

Что ещё нужно:
- поиск по содержимому сообщений;
- более явный список "чаты текущего проекта" в верхнем меню Android.

### 2. Обновление APK

Codex Desktop не имеет такого Android updater flow. Для Remote Code это критический путь, потому что пользователь часто проверяет свежую сборку через расширение.

Что исправлено сейчас:
- Android updater сначала качает APK с текущего подключенного расширения (`/api/app/apk`) с токеном;
- GitHub raw оставлен только fallback;
- внешний URL без порта используется напрямую.

Что ещё нужно:
- `/api/app/version` или manifest с `versionCode`, `versionName`, `sha256`;
- проверка SHA-256 перед установкой;
- понятный UI: "доступна версия X", "уже актуально", "скачано с ПК/с GitHub";
- запрет установки более старого APK.

### 3. Порты и внешнее подключение

Codex Desktop не просит пользователя думать о портах. Remote Code должен вести себя так же в внешнем режиме.

Что исправлено сейчас:
- Android больше не дописывает `:8799` к Keenetic/DDNS URL;
- основной экран и настройки Android больше не показывают поле "Порт";
- локальный порт остаётся внутренним значением для совместимости.

Что ещё нужно:
- QR/pairing payload из расширения: `publicUrl`, token, mode;
- кнопка "Скопировать/показать QR для телефона";
- проверка внешнего URL прямо в меню расширения до передачи в APK.

### 4. Видимость работы модели

Сырые скрытые рассуждения модели показывать нельзя. Но пользователю нужны наблюдаемые этапы: сбор контекста, ожидание модели, поток ответа, команды, правки, ошибки.

Что исправлено сейчас:
- extension генерирует события `Сбор контекста IDE`, `Ожидание модели`, `Модель ответила`, `Поток ответа`, `Ответ готов`;
- Android показывает эти события в timeline.

Что ещё нужно:
- единый визуальный формат action timeline на PC и Android;
- отдельные статусы для чтения файлов, диагностики, git diff, сборки и тестов;
- компактный "activity drawer" с фильтром: модель, команды, файлы, ошибки.

### 5. Файлы, diff и review

Codex Desktop хорошо держит связку "сообщение -> изменённые файлы -> review". В расширении это частично есть через change cards. На Android файл можно открыть, но review/diff слабее.

Что ещё нужно:
- Android diff viewer для change summary;
- кнопки "Открыть файл", "Показать diff", "Принять/отклонить patch";
- группировка файлов по проекту;
- поиск по файлам проекта.

### 6. Навигация

Codex Desktop имеет один главный рабочий поток: проект, чат, действия, файлы. Android сейчас имеет два похожих входа: `CodexScreen` и `VSCodeScreen`, плюс отдельные internal screens.

Что ещё нужно:
- сделать `CodexScreen` единственным главным экраном;
- оставить `Проекты`, `Ошибки`, `Настройки` как вторичные панели;
- убрать дублирование `VSCodeScreen`, если его функционал полностью покрыт главным экраном;
- добавить persistent project switcher рядом с названием чата.

## Рекомендуемый порядок доработок

1. P0: updater manifest + SHA-256 + "скачать с текущего расширения первым".
2. P0: project metadata для всех thread summaries, фильтр "текущий проект".
3. P1: Android project switcher в верхней панели чата. Done in Android `1.0.108`.
4. P1: единый activity timeline PC/Android.
5. P1: Android diff/review screen для change summary.
6. P2: поиск по чатам и файлам проекта. Drawer search for projects/chats done in Android `1.0.109`; message/file-content search remains.
7. P2: QR/pairing экран в расширении.
8. P2: убрать дублирующий `VSCodeScreen` после переноса функций в основной экран. Done before Android `1.0.108`.
