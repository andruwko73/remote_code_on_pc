# Remote Code on PC

Remote Code on PC связывает Android-приложение с расширением VS Code. Телефон становится удаленным интерфейсом к рабочему окну VS Code и Codex: можно вести чат, видеть ход работы модели, переключать проекты и чаты, прикреплять файлы, подтверждать действия и получать обновления приложения.

Проект состоит из двух частей:

- `extension/` - расширение VS Code с HTTP/WebSocket сервером и Codex-подобным webview.
- `android/` - Android-приложение на Kotlin и Jetpack Compose.

## Что уже сделано

### Расширение VS Code

- Codex-подобный чат в webview: история сообщений, composer, stop/run, выбор модели, profile и reasoning effort.
- Отображение хода работы модели: прогресс, последние действия, статусы команд, approve/deny и результат ответа.
- Чаты сгруппированы по проектам, а заголовки поврежденных Codex-сессий восстанавливаются из индекса истории.
- Поддержаны действия сообщений: копирование, редактирование пользовательского сообщения, удаление и повтор ответа.
- Карточка изменений показывает список файлов, additions/deletions, раскрытие списка и переход к проверке изменений.
- Вложения работают через кнопку `+` и вставку из буфера обмена.
- Настройки composer синхронизируются между расширением и Android-приложением через API/WebSocket.
- Терминальные команды не выполняются отдельным небезопасным endpoint: выполнение идет через approve/deny flow.
- Внешний HTTP/WebSocket доступ требует токен; без токена публичные status endpoints отдают только минимальную диагностику.

### Android-приложение

- Экран подключения поддерживает локальную сеть и внешний публичный URL без ручного поля порта.
- Поле токена скрыто по умолчанию, а внешний режим не запускается без токена.
- Codex-чат отображает проекты и чаты, привязанные к проектам.
- Интерфейс чата приведен ближе к Codex: компактный composer, плотные ответы, карточка изменений, toolbar сообщений и видимые шаги работы модели.
- Есть вложения, голосовой ввод, выбор profile/model/effort, stop во время генерации.
- История чата дедуплицируется после optimistic-send, WebSocket и загрузки `/api/codex/history`.
- Approve/deny действия доступны с телефона.
- Обновление APK сначала берет артефакт у подключенного расширения, проверяет SHA и подпись, затем открывает системный установщик по явному нажатию пользователя.
- Если Android требует разрешение на установку из неизвестного источника, приложение открывает системный экран разрешения и оставляет проверенный APK готовым к повторному нажатию `Установить`.
- После подтверждения установки приложение не запрашивает `return result`, чтобы системный установщик мог показать стандартный экран завершения с кнопкой `Открыть`.

### Внешнее подключение

- Основной сценарий внешнего подключения: постоянный Keenetic/KeenDNS или другой DDNS URL с прямым пробросом TCP-порта на ПК.
- Служебные/промежуточные URL не считаются рабочим публичным адресом для телефона.
- Приложение нормализует `http://...`, `https://...` и protocol-relative URL.
- Если домен резолвится, но TCP-порт недоступен или соединение обрывается до HTTP-ответа, приложение показывает отдельную понятную ошибку.
- Для внешней сети обязателен токен доступа.

## Установка расширения

Из готового VSIX:

```powershell
code --install-extension extension/remote-code-on-pc-*.vsix --force
```

После установки или обновления расширения в уже открытом VS Code выполните `Developer: Reload Window`, чтобы активное окно точно подхватило новую версию.

Команды расширения:

- `Remote Code on PC: Open Chat`
- `Remote Code on PC: Запустить сервер`
- `Remote Code on PC: Остановить сервер`
- `Remote Code on PC: Подключение`
- `Remote Code on PC: Публичный Keenetic URL`
- `Remote Code on PC: Токен доступа`

## Установка Android APK

Готовый APK лежит в репозитории:

```text
apk/app-debug.apk
```

При ручной установке Android может показать предупреждение Play Защиты или запросить разрешение на установку из неизвестного источника. Это ожидаемо для sideload APK. Разрешение нужно выдать для приложения, из которого запускается установка.

После подключения к расширению кнопка обновления в приложении использует endpoint расширения, проверяет SHA и подпись APK, показывает диалог `APK готов к установке` и только после нажатия `Установить` открывает системный установщик.

## Локальное подключение

1. Установите и запустите расширение в VS Code.
2. Убедитесь, что телефон и ПК находятся в одной сети.
3. В приложении выберите локальную сеть.
4. Укажите IP ПК.
5. Если в расширении задан токен, вставьте тот же токен в приложение.
6. Нажмите `Подключиться`.

Проверка с ПК:

```powershell
curl.exe http://127.0.0.1:8799/api/status
```

Если сервер слушает только `127.0.0.1`, телефон из локальной сети не сможет подключиться напрямую. Для подключения с другого устройства сервер должен быть доступен на сетевом интерфейсе ПК, а при любом внешнем доступе должен использоваться токен.

## Внешнее подключение

Внешний режим нужен, когда телефон не находится в одной сети с ПК.

Рекомендуемый формат публичного URL:

```text
http://name.example.net:8799
```

Требования:

- постоянный DDNS/KeenDNS домен или другой публичный домен;
- прямой проброс TCP `8799 -> IP ПК:8799`;
- запущенное расширение VS Code;
- созданный токен доступа;
- публичный URL, сохраненный в настройках расширения или введенный в приложении.

В VS Code используйте команду `Remote Code on PC: Подключение` или пункт подключения в webview. Там можно сформировать публичный URL, скопировать токен и открыть подсказки для настройки роутера.

В приложении:

1. Включите `Внешняя сеть`.
2. Вставьте публичный URL.
3. Вставьте токен.
4. Нажмите `Подключиться`.

Для проверки доступности с внешней сети:

```powershell
curl.exe -I http://name.example.net:8799/api/status
```

## Токен доступа

Токен задается настройкой VS Code:

```text
remoteCodeOnPC.authToken
```

Создать или заменить токен можно через команду `Remote Code on PC: Токен доступа`.

Если токен задан:

- HTTP-запросы должны идти с `Authorization: Bearer <token>`;
- WebSocket подключается с тем же токеном;
- Android-приложение должно использовать тот же токен.

Для внешней сети токен обязателен. Без валидного токена публичные endpoints не раскрывают приватные данные и не дают полноценный доступ к серверу.

## Приватность логов

Логи приложения и расширения маскируют:

- `Authorization: Bearer ...`;
- query-параметры с токеном;
- сохраненные токены;
- публичные URL провайдеров внешнего доступа.

Не коммитьте локальные диагностические файлы, логи, временные вложения, реальные домены, токены, пароли и абсолютные пути к файлам на своем ПК.

## Сборка

Расширение:

```powershell
cd extension
npm install
npm run compile
npm test
```

Сборка VSIX:

```powershell
cd extension
npx vsce package --allow-missing-repository
```

Android:

```powershell
cd android
./gradlew.bat :app:testDebugUnitTest :app:assembleRelease
```

После сборки release APK обновите артефакт приложения:

```powershell
Copy-Item android/app/build/outputs/apk/release/app-release.apk apk/app-debug.apk -Force
Get-FileHash apk/app-debug.apk -Algorithm SHA256
```

Запишите новый SHA-256 в `apk/app-debug.apk.sha256`.

## Проверка перед релизом

Минимальный checklist:

- `npm test` проходит полностью.
- `npm run compile` проходит без ошибок.
- `:app:testDebugUnitTest` проходит без ошибок.
- `:app:assembleRelease` собирает APK.
- `apk/app-debug.apk.sha256` совпадает с фактическим SHA-256 APK.
- Android-приложение устанавливается поверх предыдущей версии.
- Кнопка обновления APK показывает диалог установки и открывает системный установщик.
- Локальное подключение открывает экран Codex.
- Внешнее подключение проверено с публичным URL и токеном.
- В VS Code после обновления расширения выполнен `Developer: Reload Window`.

## Известные ограничения

- Внешний доступ зависит от реального роутера, DDNS/KeenDNS и проброса TCP-порта.
- Sideload APK может вызывать предупреждения Android или Play Защиты.
- После смены токена старый токен сразу перестает работать; его нужно заменить в Android-приложении.
- Старое открытое окно VS Code может продолжать работать на старой версии расширения до reload.
- Некоторые опасные действия, например удаление чата или очистка логов, требуют отдельного ручного подтверждения при проверке на живом окружении.
